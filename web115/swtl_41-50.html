Chapter 41—while Loops

While loops perform the same function as for loops but are structured differently. The limit onabort the loop is placed above the while keyword and the incrementer is placed after the statements executed during each iteration. Recall the loop from our function from Chapter 21: 

		var cars = ["Ferrari", "Lamborghini", "Porsche", "MacLaren", "Pagani"];
		for(var i=0; i<=cars.length; i++) {
			if(query===cars[i]) {
				matchFound = true;
				alert("It's on the list!");
				break;
			} 
		}
		if(matchFound===false) {
			alert("Tis not on the list.");
		}
To use a while loop we rewrite the loop as follows: 
		var cars = ["Ferrari", "Lamborghini", "Porsche", "MacLaren", "Pagani"];
		var i=0;
		while (i<=cars.length) {
			if(query===cars[i]) {
				matchFound = true;
				alert("It's on the list!");
				break;
			}
			 i++;
		}

Chapter 42—do...while Loops

The third type of loop is a do...while loop. It performs the same function as the others but is again structured differently. The counter is declared the same way as in the while loop, but the condition is placed immediately after the closing curly brace. The loop begins with the do keyword. Here is the while loop: 

		var cars = ["Ferrari", "Lamborghini", "Porsche", "MacLaren", "Pagani"];
		var i=0;
		while (i<=cars.length) {
			if(query===cars[i]) {
				matchFound = true;
				alert("It's on the list!");
				break;
			}
			i++;
		}
Here is that loop rewritten as a do...while loop: 
		var cars = ["Ferrari", "Lamborghini", "Porsche", "MacLaren", "Pagani"];
		var i=0;
		do {
			if(query===cars[i]) {
				matchFound = true;
				alert("It's on the list!");
				break;
			}
			i++;
		} while (i<=cars.length);
*** Note: the loop terminates with the semi-colon at the end of the while statement, not the closing curly brace. Also, the loop will run at least once because the statement(s) come before the condition in the code.***

See that the results are the same:

Every loop can be written in any of the three styles. The type of loop used depends on the function:
  • If you know exactly how many times the loop will run, use a for loop.
  • If you don't know how many times the loop will run, use a while loop.
  • If you need to guarantee the loop runs at least once, use a do...while loop.

Chapter 43—Placing Scripts

Scripts can be embedded in HTML or placed in an external file. Like CSS, it is usually best to store scripts in an external file in order to compartmentalize the structure from the style and the functions. If using scripts inline, it's best to place them at the end of the body section so images and styling are sorted out without interference. Scripts are enclosed with <script></script> tags within HTML. The syntax for referencing an external .js file is: 

		<script src="filename.js"><script>

Chapter 44—Commenting

Comments are used in coding to give context to sections of code. They make it easier to remember what you were doing when you revisit code written in the past and to understand your codee for other people reading it. There are two types of comments:

  • Single-line comments, which begin with // and end at the end of the line of code
  • Multiple-line comments, which begin with /* and end with */
For example, 
		// This is a single-line comment.
		
		/*
		   This is a multiple-line comment.
		*/
The browser ignores anything after // on a single line, and anything in between /* and */ over multiple lines.

Chapter 48—Events:fields

When we are working with events involving buttons/links/headings/images we use the keywords onMouseover and onMouseout to change behaviors. Fields use two different keywords to achieve the same results when the field is selected/deselected: onFocus and onBlur. We can do so using a combination of the onSubmit event handler and the document.getElementById().value method.   	<label for="last_name">Last name:</label>
  	<input type="text" id="last_name" name="last_name" value="" placeholder="Enter your last name">
  	<input class="submitButton" type="button" onclick="chap49()" value="Submit">
	</form> 
	<h3 id="greeting"></h3>
	
	function chap49() {
