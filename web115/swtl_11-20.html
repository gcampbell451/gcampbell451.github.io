Chapter 12—If...else and else if Statements

Chapter 13—Testing Sets of Conditions

The statements we examined in Chapter 12 tested for one condition only: true or false. What if we wanted to test two conditions in order for a statement to execute? We can use the operators && and/or ||. && translates to and and || translates to or.

Example:

The code is:

if (question1 < question1a && question1 < 10) {
   alert("The first number you entered is less than 10 and is smaller than the second.");
else if (question1 == question1a && question1 < 10) {
   alert("The numbers you entered are identical and both are less than 10.");
} else if (question1 == question1a && question1 > 10) {
   alert("The numbers you entered are identical and both are greater than 10.");
} else if (question1 == question1a && question1 == 10) {
   alert("The numbers you entered identical and each is 10.");
} else {
   alert("The first number you entered is greater than 10 and is larger than the second.");
}

Chapter 15—Arrays

Another type of variable we can create is an array. An array, closed in square brackets, [ ], can have multiple values assigned to it. Each value assigned to an array is also called an element.

Examples:

  • var array1 = [num1, num2, num3];
  • var array2 = ["Joe", 12, num4];

Each element in an array is assigned a value called its index. We can reference elements directly by referencing the array by its index. Let's say our array is declared as follows:

var array1 = ["red", "orange", "yellow", "green", "blue", "indigo", "violet"];

Chapter 16—Arrays: Adding and Removing Elements

Just as we can create an empty variable we can create an empty array with the following syntax:

var colors = [ ];

We then can assign values to this array by assigning them to the array's indices:

color[0] = "red";
color[1] = "orange";
color[2] = "yellow";
etc.

If we refer to an index that does not already have a value assigned the value will be undefined. If we assign a value to an index position that already contains a value, the previous value will be overwritten in memory. What if we wish to add or remove a value to the end of the array? For that task we have the pop() and push() methods.

For example, color.pop(); removes the last element of the color array. We could store this value in a variable:

var goByeBye = color.pop();

If we wish to add an item to the end of our array we use push(newElement);:

color.push("blue"); will add blue to the array we assigned values to earlier, in the last position.

Examples:

var array1 = ["red", "orange", "yellow", "green", "blue", "indigo", "violet"];

Chapter 18—for Loops

Loops automate functions that would otherwise need tedious coding. Say we want to check if a car the user enters in a prompt is in our array from the previous chapter. We assign the user's entry to a variable and iterate through the array to see if it's there. The structure of this for loop is:

	   for(var i = 0; i <= 4; i++) {
		 if(carOnList === cars[i] {
			alert("It's on the list!");
		 }
	   }
Notes:
  • The variable i counts iterations. It can be initialized to any value, but if we want to begin at the beginning of the array, it is assigned 0.
  • The second statement in the parentheses after for defines a limit on the loop. In this case, once the index counter reaches 4, the loop ends.
  • The third statement tells the browser how to increment the counter. In this case, 1 is added to i after each iteration, but it could be decremented. It could be multiplied by a number. It's up to the need of the program.

Chapter 19—for Loops: Flags, Booleans, Array Length, and Loopus Interruptus

You may have noticed a problem with the loop used in Chapter 18—if you entered a car not in the array, nothing happened. The user doesn't know if they did something wrong or if it was the function. To solve this issue we use a flag. A flag is a variable that begins with an assigned value and is then switched to another value if our conditions are met. Say we began the function with this statement:

var matchFound = false;

We then add a statement inside the loop that reassigns matchFound the value true if a match is found. 

  
	   for(var i=0; i<=4; i++){
	      if(carOnList===cars[i] {
	      matchFound = true;
	      alert("It's on the list!");
	      break;
	      }
	   }
Just below the loop structure we add another if statement:

Now, if the user enters a car not in the array, a message lets them know so. Try it:

If you were reading closely, you noticed a line in the loop that read break. This keyword is used to save computing cycles if the function finds a match early in the loop. In such a case, the break keyword halts the loop and lets the CPU get to the next instruction.

In our function we knew the exact number of elements in the array. What if we didn't? In that case, we would use the .length property of the array. We could assign:

var numElements = cars.length

and change the loop exit condition to:

i <= numElements;

This produces the same results.

Chapter 20—for Loops Nested

We can nest one loop inside another. When we do so, for each iteration of the outer loop, the inner loop runs a complete cycle of iterations. After the inner loop has completed its cycle, the outer loop increments and the inner loop starts over. This continues until the outer loop reaches its sentinel value. Let's examine this by creating a band name generator. We'll create three arrays: one for the first word of the band name, one for the second, and an empty array that will be filled with the results of our nested loop.

	   var fNames = ["Red", "Orange", "Yellow", "Green", "Blue", "Indigo", "Violet"];
	   var lNames = ["Earth", "Wind", "Fire", "Water"];
	   var bandNames = [];
	   for(var i=0; i < fNames.length; i++) {
	     for(var j=0; j < lNames.length; j++) {
		   bandNames.push(fNames[i] + lNames[j]);
		 }
	   }
	   alert(bandNames);
