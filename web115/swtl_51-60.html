Chapter 52—Manipulating Images and Text

The document.getElementById method has many uses. We can use it to change images or text. We do this by using .className after the Id for the element we wish to target. Say we have a class named "hidden" that has a property of display:none. We can hide an image or text by changing the class using this method. Here's the code inside the function: 

	document.getElementById("chap52").className = "hidden";

If we want to add a class to an image or text instead of replacing the class, we do so by using += instead of =.

Chapter 53—Swapping Images

In Chapter 47 we used mouseover to create an alert. We could have swapped images out instead with the following code: 

	<img src="before-pic.jpg" onMouseover="src='after-pic.jpg'">
We can also call a function to do this for us: 
	<img src="images/cats.jpg" id="chap53" onMouseover="chap53(id, 'images/cats2.jpg');">
	function chap53() {
				document.getElementById("chap53").src = "images/cats2.jpg";
	}
Or we could pass in the element id and the name of the image to swap as arguments, as in the below example: 
	<img src="images/cats.jpg" id="chap53" onMouseover="chap53(id, 'images/cats2.jpg');" onMouseout="chap53(id, 'images/cats.jpg');" class="center">
	
	function chap53(eId, newPic) {
		document.getElementById(eId).src = newPic;
	}

Chapter 54—Swapping Images and Setting Classes

Recall the functions from the previous chapter, where we accessed an element by its Id and replaced it. A more common approach is to assign the element to a variable. We can then combine the variable with .src to complete the function. MouseOver the picture. Here's the code: 

	function chap54(eId, newPic) {
		var a = document.getElementById(eId);
		a.src = newPic;
	}


We can use this functionality to do many things, among them change/add classes to elements stored in variables with .className as in Chapter 52.

Chapter 55—Setting Styles

Another way to alter the presentation of elements is by setting styles. We do so using the following syntax: 

		document.getElementById("id").style.[stylename] = "[value]";
For example, the following code adds a solid Carolina blue border on mouseOver of the picture below: 
	function chap55() {
		document.getElementById("chap55").style.border = "2px solid #99BADD";
	}


Chapter 56—Target All Elements by Tag Name

The method getElementById() has a cousin: getElementsByTagName(). Instead of accessing or storing a single element by its ID, we can access or store all elements in a document by their tag name. We can then use familiar properties to change their display. The code: 

	var h3 = document.getElementsByTagName("h3");
	for (var i = 0; i < h3.length; i++) {
		h3[i].style.fontSize = "3em";
	}

Chapter 58—The DOM

The DOM, or Document Object Model is an organizational chart created by the browser. Everything in the HTML5 code of a page is accounted for in this Model. The Model has a hierarchy: the top(first) level is the document, which we have accessed using document.getElementById, for example. The second level is the HTML, and the third level is made up of the head and body of the page. Below is a chart showing these relationships:







Each of the boxes in the chart are referred to as nodes. At the top level is the document node. The HTML, head, body, title, and p nodes are referred to as element nodes. At the bottom level the content within the title and p nodes are referred to as text nodes.


Chapter 59—The DOM: Parents and Children

Now that we have segmented the elements of a page into nodes, we need a way to designate the nodes. The nomenclature we use is parent and children nodes. Parent nodes enclose child nodes. Returning to the hierarchy in the chart from the previous chapter:







The document node is the parent of the html node. The head and body nodes are children of the html nodes. We refer to them, unsurprisingly, as siblings. Every node enclosed by another node is the child of that node.


Chapter 60—The DOM: Finding Children

So, now that we have a new way of identifying elements, we need a way to target them. For this, we use the keyword childNodes, followed by an array index. We can start with the document node:



To target the text element child of the 2nd child of the <div> we code: 

		var txt = document.childNodes[0].childNodes[1].childNodes[0].childNodes[1].innerHTML;
Each use of childNodes navigates down one level in the hierarchy and moves to the sibling indicated by the index number.
