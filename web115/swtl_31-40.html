Chapter 31—Getting the Current Date and Time

To get the current date and time, JavaScript creates a date object with the Date() method. It gets this information from the current system time, so the method is only as accurate as the system settings. This object is not a string so string methods will not operate upon it unless we convert it using the toString() method. 

	var dateTime = new Date();
	alert(dateTime);
What time is it?

To extract the day of the week, we use the getDay() method. This returns the day of the week as an index of an array, 0 representing Sunday through 6 representing Saturday. To convert this index to a string, a little code is necessary: 
	var days = ["Sunday", "Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday"];
	var today = new Date();
	var date = today.getDay();
	var todaysName = days[date];
	alert("The day of the week is " + todaysName + ".");

Chapter 33—Specifying a Date and Time

This chapter deals with calculating the amount of time between a reference date and another date. The strategy is to use the getTime() method for both dates and subtract them. This leaves the number of milliseconds between the two dates, and we can multiply by whatever conversion factor we want to get the desired unit for the interval. Let's find out how many days between today and the end of this year. We use a specific notation for the date: 

	var today = new Date();							// Create a date object for today.
	var endOfYear = new Date("December 31, 2020");  	// Note the naming convention on the date. This must be followed exactly.
	var msToday = today.getTime();					// Get ms since 1/1/1970 for today.
	var msEndOfYear = endOfYear.getTime();			// Get ms since 1/1/1970 for the end of this year.
	var msDiff = msEndOfYear - msToday;				// Calculate the difference.
	var days = msDiff / (1000 * 60 * 60 * 24);			// Converts ms into days.
	days = Math.floor(days);							// This rounds the above calculation down to an integer.

setDate() x.setDate(1) Day of the month is 1
setHours() x.setHours(0) Midnight
setMinutes() x.setMinutes(1) One minute past the hour

These methods change the individual elements without changing any other element. Say we have the following code: 
	var today = new Date();				// Create a Date object.
	today.setFullYear(2021);				// Change the year of the object.
	todaysYear = today.getFullYear();		// Extract the new year of the object.

Chapter 35—Functions

Functions are blocks of code that we can reuse in multiple places. Functions simplify code overall because we can call them once they are defined instead of rewriting the code each time. Yes, we could copy/paste code, but this would make the code more tedious to read and would require more storage space.

This has been happening in the background every time you have clicked a button on these pages. The button simply calls the function, which is stored in a JavaScript file. For example, the code from the previous chapter was displayed as this: 
	var today = new Date();				// Create a Date object.
	today.setFullYear(2021);				// Change the year of the object.
	todaysYear = today.getFullYear();		// Extract the new year of the object.
This code was simply wrapped in a function call in the separate file. In that file, it looks like this: 
	function chap34() {
		var today = new Date();				// Create a Date object.
		today.setFullYear(2021);				// Change the year of the object.
		todaysYear = today.getFullYear();		// Extract the new year of the object.
	}
The function keyword denotes the beginning of the function. The name of this function is chap34(). Functions names follow the same rules as variable names. The parenthesis either enclose parameters or are empty if the function does not receive them. Once called, the statements inside the curly braces are executed in order. When the ending curly brace is reached, the next line of code following it is executed. Functions are called in code like this: 
	chap34();
They can also be used in events (such as button clicks, as we have used them in these pages) like this: 
		<button class="swtlbutton" onclick="chap34();">Let's Do the Time Warp Now!</button>

Chapter 36—Functions: Passing Them Data

All of the functions we have used to this point have had no arguments. That is, we have not passed any data into the functions. If we wish to pass data into a function we do so with parameters. These parameters can be strings, numbers, or variables and are listed, separated by commas, within the parentheses in the function statement: 

	function functionName(parameter1, parameter2,....) {
	     statements;
	}
When calling the function, we pass arguments into the function. The arguments are passed into the function according to the order they are listed. 
	functionName(argument1, argument2,...);
This function passes arguments entered by the user into an internal function that sums the numbers: 
	function chap36() {
		var userNum1 = "Enter a number";
		var defaultAnswer1 = "Enter number here";
		var userNum2 = "Enter another number";
		var defaultAnswer2 = "Enter number here";
		var num1 = parseInt(prompt(userNum1, defaultAnswer1));
		var num2 = parseInt(prompt(userNum2, defaultAnswer2));
		function sum(num1, num2) {
			var sum = num1 + num2;	
			alert("The sum of " + num1 + " and " + num2 + " is " + sum + ".");
		}
		sum(num1, num2);
	}

Chapter 37—Functions: Passing Data Back from Them

We can also pass data back from functions using the return keyword. This passes back a single value from the function. We capture this value with a variable and use it as we wish. Using the code from the previous chapter, we alter the interior function as follows: 

	function sum(num1, num2) {
		return num1 + num2;			// Use return keyword to return the sum
	}
	var total = sum(num1, num2);			// Capture the returned value of the sum function
	alert("The sum of " + num1 + " and " + num2 + " is " + total + ".");
*** Note that we no longer need to call sum. Its returned value was passed to chap37() when we assigned it to total .***

Chapter 38—Functions: Local Vs. Global Variables

Local and global variables differ in their scope. Scope refers to where the variable can be seen by the code. A global variable is declared in the main code whereas a local variable is declared within a function. This local variable cannot be seen outside the function and has no meaning except inside the function. For example: 

	function chap38() {
		var global = "Global Scope";
		function chap38a() {
			var local = "Local Scope";
		}
		alert(local);
	}
Because local is declared inside the inner function it has no meaning to the outer function.

Chapter 39—Switch Statements: How to Start Them

Switch statements are an alternative to if statements when we need to test multiple conditions. An important difference is we cannot use inequalities in the conditions. The general structure is: 

	switch(variable) {
	case [condition]:
	  statement;
	  break;
	}
The break keyword exits the switch if the condition is met; otherwise the switch moves to the next condition to test. Here is a specific example: 
	switch(num) {
	case 1:
	  alert("You chose the number one.");
	  break;
	case 2:
	  alert("You chose the number two.");
	  break;
	case 3:
	  alert("You chose the number three.");
	  break;
	case 4:
	  alert("You chose the number four.");
	  break;
	}
