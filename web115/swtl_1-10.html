Chapter 4—Variables Names Legal and Illegal

It shouldn't be surprising that there are firm naming conventions for variables as computers are literal to a fault. Here are the main rules:

  • The variable name can't be enclosed in quotation marks.
  • The name can't be a number or start with a number.
  • JavaScript keywords can't be used.
  • The name can't contain spaces.
  • The name must consist of only letters, numbers, dollar signs, and underscores.
  • The name can contain JavaScript keywords.
  • Capital letters are allowed but JavaScript is case-sensitive.

Examples:

Legal Names

  • name
  • name1
  • name_1$
  • firstName
  • newAlert

Illegal Names

  • "name"
  • 2name
  • alert
  • first Name
  • first%Name

Chapter 5—Math Expressions: Familiar Operators

We saw earlier that we can write var num = 1974; to store the number 1974 in memory. We could alternatively have written var num = 1900 + 74; to store this number. When JavaScript sees a mathematical expression, it does the math. As before, we can mix variables and numbers, or use only variables.

Examples:

  • var num = 1900;
    alert(num + 74);
  • var num = 1900;
    alert(num + num);

Chapter 6—Math Expressions: Unfamiliar Operators

To increment a variable, we can write num = num + 1; however, a shorthand for this operation is num++;. We do need to be careful using this operator, though, because the literal tendencies of computers get in the way. If we wrote

var num = 1;
var newNum = num++;

we might assume that both variables are incremented. However, this is not the case, because the value of num was already stored in memory before the incrementation. To ensure both variables are incremented, put the incrementation operator before the variable.

Examples:

  • var num = 1;
    var newNum = num++;
  • var num = 1;
    var newNum = ++num;

Chapter 7—Math Expressions: Eliminating Ambiguity

When an expression has more than two operands, the order of operations becomes very important. The expression
2 + 2 * 4 evaluates differently depending on whether we multiply first or add first. JavaScript has precedence rules that state multiplication and division are performed first, then addition and subtraction. It is better to specify the order of operations for clarity using parentheses.

Chapter 9—Prompts

When we want to ask the user for information, we use a prompt box. The prompt() function takes either one or two arguments:

prompt(question, [default answer]);

The user's response is a text string, even if the response is a number. As with alert boxes, the arguments can be strings or variables and the formal code is

window.prompt();

Examples:

  • var question = prompt("Do you like dogs?", "Yes!");
